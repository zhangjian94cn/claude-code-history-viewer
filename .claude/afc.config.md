## Key Patterns & Checklist

### Security
- User input IDs for file paths: validate with regex `^[A-Za-z0-9_-]+$`
- File writes: temp file + rename pattern (atomic, safe on Windows)
- Directory traversal: `walkdir` with symlink checks
- Symlink detection: `fs::symlink_metadata()` before operations

### Error Handling
- **Frontend**: All `async/await` must have `try/catch` + user-facing toast/alert feedback
- **Backend**: Errors propagate via `Result<T, String>`, converted to IPC responses
- **Multi-step saves**: Validate + parse all data before committing state
- **Required params**: Guard checks at function entry for `projectPath`, file paths

### i18n
- New keys: Add to **all 5** language files simultaneously
- No duplicate keys in same namespace file
- All user-visible strings: wrap in `t()`
- Validation: Run `pnpm run i18n:validate` before commit

### Accessibility (a11y)
- Icon-only buttons: must have `aria-label`
- Dialogs: require `DialogTitle` or `aria-label`
- Label-Input pairs: use `htmlFor` + `id` (use `React.useId()` for unique IDs)
- TooltipTrigger: wrap in focusable element like `<button>`
- Virtual lists: preserve semantic structure for screen readers

### React State Management
- **Never read state immediately after setState**: pass value as argument or use `useEffect`
- **Custom hooks inside custom hooks**: careful with instance sharing
- **Zustand slices**: keep slices focused (one concern per slice)

### Cross-Platform
- Path splitting: `split(/[\\/]/)` to handle both Windows `\` and Unix `/`
- Rust `fs::rename`: Windows fails if target exists → use `remove_file` first
- Home directory detection: support `C:\Users\` pattern on Windows
- Env vars: `HOME` not reliable; use `dirs::home_dir()` instead

### Performance
- Virtual scrolling: VariableSizeList for message lists (100+ items)
- Message memoization: wrap in React.memo()
- Height caching: pre-calculate and cache item heights
- SIMD-JSON: use for large JSONL parsing in Rust backend
- Memmap: memory-mapped file access for large files

### Other Conventions
- Utility functions: check existing utils before writing (avoid duplication)
- null checks: use `!= null` (loose equality) to catch both null and undefined
- localStorage access: always wrap in try/catch (can fail in private browsing)
- Tauri IPC: all commands are async; Promises, not callbacks
